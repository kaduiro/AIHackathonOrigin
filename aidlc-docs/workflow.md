# AI-DLC 軽量準拠ワークフロー

## 概要
AI-DLC（AI-Driven Development Life Cycle）の考え方を軽量に採用し、ハッカソン規模のMVP開発に適用する。

## 基本原則
- 各段階で承認ゲート（ユーザー確認）を設ける
- 質問は一度にひとつ、選択肢を番号付きで提示
- 会話ログは `/interaction_logs/yyyymmdd.jsonl` に記録

---

## フェーズ構成

```mermaid
flowchart LR
    A[1. 要件確認] --> B[2. システム設計]
    B --> C[3. ユニット分割]
    C --> D[4. 実装ループ]
    D --> E[5. ビルド・テスト]
```

### Phase 1: 要件確認（INCEPTION 軽量版）
- **入力**: 要件定義書（PRD）
- **作業**: 不明点の洗い出し、技術選定の確認
- **成果物**: `requirements-confirmed.md`
- **承認ゲート**: ユーザーが要件確認完了を承認

### Phase 2: システム設計
- **作業**: アーキテクチャ設計、コンポーネント設計、DB設計、API設計
- **成果物**:
  - `architecture.md` — 全体アーキテクチャ
  - `database-design.md` — DB スキーマ
  - `api-design.md` — API エンドポイント一覧
  - `components.md` — フロントエンドコンポーネント構成
- **承認ゲート**: ユーザーが設計を承認

### Phase 3: ユニット分割
- **作業**: 実装順序に沿ったユニット（作業単位）の定義
- **成果物**: `units-of-work.md`
- **承認ゲート**: ユーザーがユニット分割を承認

### Phase 4: 実装ループ（CONSTRUCTION）
各ユニットについて以下を繰り返す:
1. ユニットの機能設計（必要な場合）
2. コード生成
3. 動作確認
4. 次のユニットへ

### Phase 5: ビルド・テスト
- **作業**: 全体ビルド、結合テスト、動作確認
- **成果物**: 動作するMVP
